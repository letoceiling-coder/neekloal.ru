# UX-спецификация: neeklo-ai-path

Документ описывает пользовательский опыт и визуально-поведенческие паттерны эталонного фронтенда из репозитория [Neeklo1606/neeklo-ai-path](https://github.com/Neeklo1606/neeklo-ai-path) (Vite + React + Tailwind + shadcn/ui). Цель — воспроизвести тот же уровень ясности навигации, иерархии экранов и «студийного» монохромного UI при переносе или сравнении с текущим монорепозиторием.

---

## 1. Продуктовая модель

- **Позиционирование:** AI-продакшн студия (ролики, сайты, Telegram Mini App, AI-агенты), заказ через чат и каталог услуг.
- **Ключевой сценарий:** лендинг → чат с AI-сбором брифа → карточка брифа → коммерческое предложение → при необходимости переход к чату с менеджером.
- **Вторичные сценарии:** каталог услуг и кейсов, проекты пользователя, профиль, настройки, юридические страницы, уведомления.

---

## 2. Информационная архитектура и маршруты

| Маршрут | Назначение |
|--------|------------|
| `/` | Лендинг (hero, портфолио, услуги, шаги, CTA) |
| `/login`, `/register` | Вход и регистрация |
| `/chat` | Основной AI-чат (сценарий брифа и КП) |
| `/services`, `/services/:slug` | Список услуг и детальная страница |
| `/cases` | Кейсы |
| `/projects`, `/projects/:id` | Список проектов и карточка проекта |
| `/order/:serviceId` | Оформление заказа по услуге |
| `/profile`, `/settings` | Профиль и настройки |
| `/manager-chat` | Чат с менеджером (после сценария в `/chat`) |
| `/notifications` | Уведомления |
| `/legal/:slug` | Юридические документы |
| `*` | Страница 404 |

Переходы между страницами оборачиваются в лёгкую анимацию появления (`PageTransition` → класс `animate-page-in` в стилях).

---

## 3. Оболочка приложения (shell)

### Десктоп (от `md` и выше)

- **Верхняя панель:** `sticky`, полупрозрачный фон с размытием (`backdrop-blur-xl`), нижняя граница, контент ограничен по ширине (~`max-w-[1200px]`), высота ~64px.
- **Бренд слева:** квадратная иконка с логотипом (Sparkles), подпись «neeklo».
- **Навигация по разделам:** Главная, Чат, Услуги, Кейсы, Проекты; отдельно — поиск (открывает полноэкранный/оверлейный поиск), колокольчик уведомлений (с индикатором точки), Профиль.
- **Активный пункт:** заливка `primary`, неактивный — `muted-foreground` с hover на `foreground` / `muted`.

### Мобильный

- **Нижняя навигация:** фиксированная панель (`fixed-bottom-nav`), размытие, верхняя граница, учёт `safe-area-inset-bottom`.
- **Четыре пункта:** Главная, Чат, Проекты, Профиль (урезанный набор относительно десктопа).
- **Минимальная зона нажатия:** ориентир ~44px по высоте, иконки ~22px, подписи ~10px, жирнее шрифт у активного пункта.

Страницы с длинным контентом используют нижний отступ (`page-container` с `padding-bottom`), чтобы контент не перекрывался нижней навигацией на мобильных.

## 4. Первый запуск: онбординг

- Показывается, если в `localStorage` нет ключа `neeklo_onboarded`.
- **Три экрана** с иконкой, заголовком, текстом (многострочным), основной кнопкой и стрелкой; внизу — **индикатор шагов** (точки).
- При переходе между шагами контент слегка **исчезает и сдвигается вниз** (короткая анимация ~250ms), затем показывается следующий экран.
- На последнем шаге по нажатию выставляется `neeklo_onboarded` и онбординг закрывается.

Смысловые блоки (по коду): знакомство с neeklo как AI-студией, акцент на чате, работа с проектами и командой.

---

## 5. Главная страница (лендинг)

- **Фон hero:** компонент «плазмы» (Plasma) + градиентный оверлей для читаемости текста.
- **Верхняя зона:** бейдж вроде «AI-продакшн студия · Москва», крупный заголовок с акцентом на скорости (например «за 48 часов»), подзаголовок про AI-ролики, сайты, Mini App и автоматизацию.
- **Действия:** первичная кнопка «Заказать проект» → `/chat`; вторичная — плавный скролл к блоку работ.
- **Социальное доказательство:** блок рейтинга (например «4.9 / 5») и анимированные счётчики при появлении в зоне видимости (`useScrollReveal`, `useCountUp`).
- **Галерея работ:** сетка карточек с изображениями, тегами (AI-видео, Сайт, Mini App и т.д.), при клике — переход в проекты; на мобильных возможен `active:scale` для тактильной отдачи.
- **Каталог направлений (продукты):** карточки в стиле `game-card` с бейджами (ХИТ, ТОП), названием, ценой «от …», кратким описанием; переход на `/services/:slug`.
- **«Как это работает»:** три шага (описать задачу → AI собирает бриф → результат и менеджер), появление блоков с задержкой при скролле.
- **Финальный CTA:** призыв к бесплатной консультации, кнопка в чат.
- **Подвал:** отдельный компонент `Footer` с ссылками и информацией.
- На мобильных внизу дублируется доступ к основным разделам через `BottomNav`.

---

## 6. Чат с AI (`/chat`)

- **Структура разговора:** чередование сообщений пользователя и AI, вставки **чипов** быстрых ответов, индикатор **печати** перед ответом AI, затем встраиваемые карточки **брифа** и **коммерческого предложения**.
- **Сценарий по шагам (пример из кода):**
  1. AI приветствует и предлагает выбрать тип задачи (чипы: лендинг, приложение, магазин, другое).
  2. Уточнение бюджета (чипы с диапазонами).
  3. Сроки (чипы).
  4. После прохождения шагов — сообщение о готовом брифе и показ **BriefCard**.
  5. После утверждения брифа — **ProposalCard** с КП.
- **Ввод:** на десктопе поле внизу экрана с отправкой по Enter; на мобильном — компонент `ChatInput` (нижняя зона под безопасную область).
- **Переход к менеджеру:** действие «подключить менеджера» добавляет сообщения и через короткую задержку ведёт на `/manager-chat`.

Визуально: пузыри `.message-bubble-user` / `.message-bubble-ai`, чипы `.chip-button`, анимации появления сообщений, чипов, брифа и предложения (см. раздел 8).

## 7. Визуальная система

### Типографика и шрифт

- Основной шрифт: **Inter** (Google Fonts), начертания 400–800.
- Заголовки лендинга и заголовки страниц — крупные, плотные (`font-bold` / `font-extrabold`), межстрочные интервалы «воздушные» для подзаголовков.

### Цвет и тема

- База: **shadcn default**, `baseColor: slate`, **CSS-переменные** в `:root`.
- Светлая монохромная схема: чёрный `primary` на белом фоне, серые `muted`, красный акцент для деструктивных действий и индикаторов (например точка на колокольчике).
- Скругления: `--radius: 0.875rem`, карточки и кнопки согласованы (`rounded-xl` / `12px` для кнопок в компонентных классах).

### Компоненты UI

- Набор **shadcn/ui**: тосты (`Toaster`, Sonner), `TooltipProvider`, прочие примитивы в `components/ui`.
- Кастомные блоки: `SearchOverlay`, карточки проектов/предложений, скелетоны загрузки.

### Адаптивность

- Контрольная точка **768px** (`md`): переключение между нижней навигацией и десктоп-хедером, изменение отступов контента (`page-content` max-width 430px → 800px → 960px на очень широких экранах).

---

## 8. Микроанимации и отклик

- **Страница:** лёгкий `translateY` + fade при входе.
- **Hero:** поэтапное появление строк (`hero-line`, `hero-fade`, `hero-scale`).
- **Скролл-ревил:** блоки работ, продуктов и шагов проявляются с задержкой по индексу (stagger).
- **Чат:** появление сообщений, «пружинистое» появление чипов, плавное появление брифа/КП, три точки печати с bounce.
- **Кнопки и карточки:** `active:scale-[0.97]` (или близко) для ощущения нажатия на тач-устройствах.

---

## 9. Глобальные уведомления и подсказки

- Два канала тостов: классический и Sonner — для обратной связи по действиям.
- `TooltipProvider` оборачивает приложение для единообразных подсказок на интерактивных элементах, где это уместно.

---

## 10. Критерии соответствия при реализации «как в репозитории»

1. Сохранить **разделение мобильной нижней навигации и десктопного хедера** с тем же набором ссылок (с учётом того, что на мобильных нет прямых пунктов «Услуги» и «Кейсы» в bottom bar — доступ через лендинг/поиск/меню).
2. Сохранить **последовательность онбординга** и ключ `localStorage` для одноразового показа.
3. Воспроизвести **лендинг**: plasma-hero, галерея, карточки услуг, три шага, CTA, footer.
4. В чате — **типизированный сценарий** с чипами, брифом, КП и ответвлением на менеджера.
5. Использовать **те же дизайн-токены** (монохром, Inter, радиусы, классы `.btn-primary`, `.game-card`, пузыри сообщений).

## 11. Зависимости стека (справочно)

Репозиторий ориентирован на: React, React Router, TanStack Query, Tailwind CSS, shadcn/ui (Radix), Lucide-иконки, Supabase в `supabase/` для бэкенда — без изменения UX на уровне макета при условии тех же экранов и состояний.
