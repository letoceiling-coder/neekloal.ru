AI Assistants API v1

REST API для интеграции AI-ассистентов в любые приложения — сайты, мобильные приложения, Telegram-боты, CRM-системы.

ℹ️

Base URL: все эндпоинты доступны по адресу https://your-domain.ru/api/v1

Аутентификация

Каждый запрос должен содержать API ключ. Ключ передаётся одним из двух способов:

Способ 1 — заголовок Authorization

HTTP Header 
Authorization: Bearer sk-live_xxxxxxxxxxxxxxxxxxxxxxxx

Способ 2 — заголовок X-Api-Key

HTTP Header 
X-Api-Key: sk-live_xxxxxxxxxxxxxxxxxxxxxxxx
⚠️

API ключи начинаются с префикса sk-. Не путайте с JWT-токенами дашборда — они предназначены только для внутреннего использования.

Где получить ключ

API ключи создаются в разделе Настройки → API ключи дашборда. При создании ключа можно указать:

  • Название ключа (для идентификации)
  • Привязка к конкретному ассистенту (опционально)
  • Разрешённые домены (опционально, для виджетов)

POST /chat — обычный чат

POST https://your-domain.ru/api/v1/chat

Отправить сообщение ассистенту и получить полный ответ в формате JSON. Если conversationId не передан — автоматически создаётся новый диалог, его id возвращается в ответе.

Тело запроса (application/json)

ПараметрТипОбязательностьОписание
agentId string (UUID) обязательный

UUID ассистента из дашборда
message string обязательный

Текст сообщения пользователя
conversationId string (UUID) опциональный

ID существующего диалога для продолжения контекста. Если не передан — создаётся новый.
model string опциональный

Переопределить LLM модель, например qwen2.5:7b, llama3.1:8b. Приоритет: запрос → ассистент → система.
temperature number опциональный

Температура генерации (0.0 – 2.0). Выше — креативнее, ниже — точнее.
maxTokens number опциональный

Максимальное число токенов в ответе.

Ответ 200 OK

JSON 
{
  "reply":          "Привет! Чем могу помочь?",
  "conversationId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "model":          "qwen2.5:7b",
  "tokens":         { "prompt": 42, "completion": 18, "total": 60 },
  "created":        true   // true если диалог был создан автоматически
}

Описание полей ответа

ПолеТипОписание

ID диалога — сохраните для следующих сообщений

POST /chat/stream — стриминг (SSE)

POST https://your-domain.ru/api/v1/chat/stream

Потоковый ответ через Server-Sent Events (SSE). Токены приходят по мере генерации — идеально для "живого" отображения ответа в UI.

ℹ️

Тело запроса идентично POST /chat. Параметры: agentId, message, conversationId, model, temperature, maxTokens.

Заголовки ответа

HTTP Response Headers 
Content-Type:    text/event-stream; charset=utf-8
Cache-Control:   no-cache, no-transform
Connection:      keep-alive
X-Accel-Buffering: no

SSE события

token

Очередной токен ответа. Приходит много раз в процессе генерации.

SSE
event: token
data: {"token": "Привет"}

event: token
data: {"token": "!"}
done

Генерация завершена. Последнее событие в потоке.

SSE
event: done
data: {
  "conversationId": "a1b2c3d4-...",
  "model":          "qwen2.5:7b",
  "tokens":         {"prompt":42, "completion":18},
  "created":        true
}
error

Ошибка во время генерации (таймаут 60s, ошибка AI-бэкенда и т.п.).

SSE
event: error
data: {"error": "Stream timeout (60 s)"}

javascript 
async function chat(agentId, message, conversationId = null) {
  const res = await fetch("https://your-domain.ru/api/v1/chat", {
    method: "POST",
    headers: {
      "Authorization": `Bearer sk-live_xxxxxxxxxxxxxxxx`,
      "Content-Type":  "application/json",
    },
    body: JSON.stringify({ agentId, message, conversationId }),
  });

  if (!res.ok) {
    const err = await res.json();
    throw new Error(err.error);
  }

  return res.json();
  // { reply, conversationId, model, tokens, created }
}

// Использование:
const { reply, conversationId } = await chat(
  "11111111-2222-3333-4444-555555555555",
  "Привет! Что ты умеешь?"
);
console.log(reply);

async function streamChat(agentId, message, onToken, onDone) {
  const res = await fetch("https://your-domain.ru/api/v1/chat/stream", {
    method: "POST",
    headers: {
      "Authorization": `Bearer sk-live_xxxxxxxxxxxxxxxx`,
      "Content-Type":  "application/json",
    },
    body: JSON.stringify({ agentId, message }),
  });

  const reader  = res.body.getReader();
  const decoder = new TextDecoder();
  let buffer = "";

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });

    const lines = buffer.split("\n");
    buffer = lines.pop();  // сохраняем незавершённую строку

    for (const line of lines) {
      if (line.startsWith("event: token")) continue;
      if (line.startsWith("data: ")) {
        const data = JSON.parse(line.slice(6));
        if (data.token)          onToken(data.token);
        if (data.conversationId) onDone(data);
      }
    }
  }
}

// Использование:
let fullText = "";
await streamChat(
  "11111111-2222-3333-4444-555555555555",
  "Напиши стихотворение",
  (token) => { fullText += token; console.log(token); },
  (done)  => { console.log("Готово, модель:", done.model); }
);
typescript 
class AssistantClient {
  private baseUrl: string;
  private apiKey: string;
  conversationId: string | null = null;

  constructor(baseUrl: string, apiKey: string) {
    this.baseUrl = baseUrl;
    this.apiKey  = apiKey;
  }

  private headers() {
    return {
      "Authorization": `Bearer ${this.apiKey}`,
      "Content-Type":  "application/json",
    };
  }

  async send(agentId: string, message: string) {
    const res = await fetch(`${this.baseUrl}/api/v1/chat`, {
      method: "POST",
      headers: this.headers(),
      body: JSON.stringify({
        agentId,
        message,
        conversationId: this.conversationId,
      }),
    });
    const data = await res.json();
    if (!res.ok) throw new Error(data.error);
    this.conversationId = data.conversationId;  // сохраняем контекст
    return data.reply as string;
  }

  async *stream(agentId: string, message: string) {
    const res = await fetch(`${this.baseUrl}/api/v1/chat/stream`, {
      method: "POST",
      headers: this.headers(),
      body: JSON.stringify({ agentId, message, conversationId: this.conversationId }),
    });
    const reader  = res.body!.getReader();
    const decoder = new TextDecoder();
    let buf = "";
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      buf += decoder.decode(value, { stream: true });
      const lines = buf.split("\n"); buf = lines.pop()!;
      for (const line of lines) {
        if (!line.startsWith("data: ")) continue;
        const d = JSON.parse(line.slice(6));
        if (d.token)          yield d.token as string;
        if (d.conversationId) this.conversationId = d.conversationId;
      }
    }
  }
}

// Использование:
const client = new AssistantClient(
  "https://your-domain.ru",
  "sk-live_xxxxxxxxxxxxxxxx"
);
const AGENT = "11111111-2222-3333-4444-555555555555";

// Сессия с контекстом:
console.log(await client.send(AGENT, "Привет!"));
console.log(await client.send(AGENT, "А что умеешь?"));  // помнит контекст

import requests

API_URL   = "https://your-domain.ru/api/v1"
API_KEY   = "sk-live_xxxxxxxxxxxxxxxx"
AGENT_ID  = "11111111-2222-3333-4444-555555555555"

headers = {

conversation_id = data["conversationId"]

# Продолжаем тот же диалог
resp2 = requests.post(
    f"{API_URL}/chat",
    headers=headers,
    json={
        "agentId":        AGENT_ID,
        "conversationId": conversation_id,
        "message":        "А что ты умеешь?",
    },
)
print(resp2.json()["reply"])
python 
import requests, json

API_URL   = "https://your-domain.ru/api/v1"
API_KEY   = "sk-live_xxxxxxxxxxxxxxxx"
AGENT_ID  = "11111111-2222-3333-4444-555555555555"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type":  "application/json",
}

with requests.post(
    f"{API_URL}/chat/stream",
    headers=headers,
    json={"agentId": AGENT_ID, "message": "Напиши стихотворение"},
    stream=True,
) as resp:
    event_type = None
    for raw_line in resp.iter_lines(decode_unicode=True):
        if raw_line.startswith("event: "):
            event_type = raw_line[7:]
        elif raw_line.startswith("data: "):
            data = json.loads(raw_line[6:])
            if event_type == "token":
                print(data["token"], end="", flush=True)
            elif event_type == "done":
                print()  # новая строка после завершения
                print(f"Модель: {data['model']}, токены: {data['tokens']}")
            elif event_type == "error":
                print(f"Ошибка: {data['error']}")

Встраивание виджета

Для встраивания чата на сайт используйте виджет — он автоматически обращается к API через API ключ с ограничением по домену.

html 
<!-- Вставьте перед </body> -->
<script
  src="https://your-domain.ru/widget.js"
  data-agent-id="11111111-2222-3333-4444-555555555555"
  data-api-key="sk-live_xxxxxxxxxxxxxxxx"
  defer
></script>
🔐

Для виджетов создайте отдельный API ключ с ограничением по домену — так ключ нельзя будет использовать с других сайтов.

Ограничение по доменам

API ключ можно ограничить списком разрешённых доменов. Это защищает ключ от использования с посторонних сайтов.

ПаттернЧто разрешает
example.com

Только точный домен example.com

*.example.com

Любые поддомены: shop.example.com, app.example.com

(пусто)

Запросы с любого домена (и без Origin)

При запросе с запрещённого домена API вернёт:

HTTP 403 
{
  "error": "Domain \"evil.com\" is not allowed for this API key"
}
⚠️

Если домены указаны, то в запросе обязательно должен быть заголовок Origin или Referer — иначе сервер не сможет проверить домен и вернёт 403.
